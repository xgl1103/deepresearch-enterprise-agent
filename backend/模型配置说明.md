# 模型配置说明

## 如何添加/修改可用模型

### 方法：修改 .env 配置文件

编辑 `backend/.env` 文件中的 `AVAILABLE_MODELS` 环境变量。

### 配置格式

```env
AVAILABLE_MODELS=[{"model_id":"模型ID","display_name":"显示名称","icon":"图标类型","icon_color":"颜色类名"}]
```

**字段说明：**
- `model_id`: 实际调用LLM时使用的模型标识
- `display_name`: 前端下拉框显示的名称
- `icon`: 图标类型，可选 `Zap`（闪电）或 `Cpu`（芯片）
- `icon_color`: 图标颜色，使用 Tailwind CSS 颜色类名

### 示例：添加新模型

在 `.env` 文件中添加或修改：

```env
AVAILABLE_MODELS=[{"model_id":"qwen3.6-flash","display_name":"Qwen-Flash","icon":"Zap","icon_color":"yellow-400"},{"model_id":"qwen3.6-plus","display_name":"Qwen-Plus","icon":"Zap","icon_color":"orange-400"},{"model_id":"qwen3.7-max","display_name":"Qwen-Max","icon":"Cpu","icon_color":"purple-400"},{"model_id":"gpt-4o","display_name":"GPT-4o","icon":"Cpu","icon_color":"green-400"},{"model_id":"claude-3.5","display_name":"Claude 3.5","icon":"Cpu","icon_color":"blue-400"}]
```

**范例：**
```json
[
  {"model_id":"qwen3.6-flash","display_name":"Qwen-Flash","icon":"Zap","icon_color":"yellow-400"},
  {"model_id":"qwen3.6-plus","display_name":"Qwen-Plus","icon":"Zap","icon_color":"orange-400"},
  {"model_id":"qwen3.7-max","display_name":"Qwen-Max","icon":"Cpu","icon_color":"purple-400"},
  {"model_id":"gpt-4o","display_name":"GPT-4o","icon":"Cpu","icon_color":"green-400"},
  {"model_id":"claude-3.5","display_name":"Claude 3.5","icon":"Cpu","icon_color":"blue-400"}
]
```

### 图标选项

- **Zap**: 闪电图标（适合快速模型）
- **Cpu**: 芯片图标（适合强大模型）

### 颜色选项

使用 Tailwind CSS 颜色类，例如：
- `yellow-400`, `orange-400`, `red-400`
- `green-400`, `blue-400`, `purple-400`
- `pink-400`, `indigo-400`, `teal-400`

### 生效步骤

1. 修改 `backend/.env` 文件中的 `AVAILABLE_MODELS`
2. 重启后端服务
3. 刷新前端页面，模型列表自动更新

### 注意事项

- JSON 格式必须正确，注意引号和逗号
- 确保 `model_id` 与后端 LLM 服务支持的模型名称一致
- 如果未配置 `AVAILABLE_MODELS` 或格式错误，系统会使用默认的3个通义千问模型
- 修改后无需重新编译前端代码
- 前端会在 API 请求失败时使用默认模型列表作为降级方案
